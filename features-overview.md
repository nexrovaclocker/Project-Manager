# Nexrova Project Manager - Features Overview

This document outlines all the user-facing features available in the application, broken down into non-technical terms.

## 1. User Accounts and Roles
* **Secure Login:** Users can log into the platform securely using their unique username and password.
* **Role-Based Access:** The system supports different types of users:
  * **Admin:** Has full access to the platform, including a dedicated Admin Panel with special administrative controls for managing the team and system operations.
  * **Member:** Standard access for team members to collaborate and track their work.
  * **Intern:** Limited access tailored for interns.

## 2. Dashboard and Navigation
* **Central Hub:** A clean, organized dashboard that acts as the main workspace.
* **Intuitive Tabs:** Easy navigation between different sections like "Overview", "Projects", and "Daily Stats".

## 3. Time Tracking (Clock Panel)
* **Clock In / Clock Out:** Users can easily punch in when they start working and punch out when they finish.
* **Session Details:** The system automatically calculates the duration of each work session.
* **Session Notes:** Users can attach specific notes or comments to their individual clock-in sessions to explain what they worked on.

## 4. Collaborative Notes System
* **Shared Notes:** Users can create and share notes with their team.
* **Dynamic Blocks:** Notes aren't just plain text; they can be built using different blocks, including regular text paragraphs and interactive to-do lists.

## 5. Task and To-Do Management
* **Personal & Shared Tasks:** Users can create to-do lists within their notes or directly inside projects.
* **Accountability:** When a task is checked off as complete, the system records exactly *who* completed it, ensuring accountability across the team.

## 6. Project Management
* **Project Creation:** Users can create new projects, giving them a specific name and description.
* **Team Assignment:** Specific team members can be added to or removed from individual projects, restricting or granting access as needed.
* **Project-Specific Tasks:** Each project has its own dedicated list of to-dos and tasks that the assigned members can collaborate on.

## 7. Daily Logs
* **Work Reports:** Users have a dedicated panel to write and submit daily logs. This allows team members to summarize their daily accomplishments or blockers.
* **Historical Tracking:** These logs are safely stored with the date and time they were created.

## 8. Statistics and Insights (Daily Stats)
* **Activity Overview:** A dedicated panel to view daily statistics, keeping everyone informed of their progress, hours logged, and completed tasks.
